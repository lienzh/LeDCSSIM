# DCS仿真组态平台 - 功能开发规范

> 本文档用于指导AI编程助手实现DCS仿真组态软件中的三项核心功能。
> 每个功能包含：功能描述、数据模型、交互规范、边界条件和验收标准。
> 请在开发每个功能前完整阅读对应章节。

## 全局数据模型

整个组态平台的核心数据结构如下，三个功能共享这套模型：

```typescript
// ===== 项目级 =====
interface DCSProject {
  projectId: string
  pages: PageMeta[]          // 所有组态页面的索引
  globalIOList: IOPoint[]    // 全厂IO点总表
}

// ===== 页面级 =====
interface PageMeta {
  pageId: string
  pageName: string           // 如 "锅炉给水泵联锁"
  area: string               // 工艺区域，如 "锅炉系统"
  tags: string[]             // 该页面涉及的核心tag列表（用于快速索引）
}

interface ConfigPage extends PageMeta {
  blocks: FunctionBlock[]
  connections: Connection[]
  ioPoints: IOPoint[]
}

// ===== 功能块 =====
interface FunctionBlock {
  blockId: string            // 如 "FB-0042"
  blockType: string          // 如 "AND" | "OR" | "PID" | "TIMER" | "SELECT" | "COMPARATOR" ...
  label: string              // 用户给的显示名
  position: { x: number; y: number }
  ports: Port[]
}

interface Port {
  portId: string
  name: string               // 如 "IN1" "OUT" "SP" "PV" "MV"
  direction: 'input' | 'output'
  signalType: 'AI' | 'AO' | 'DI' | 'DO' | 'PULSE' | 'INTERNAL'
  connectedTo: string | null // 连接目标的 portId 或 null
  dataType: 'float' | 'bool' | 'int'
}

// ===== IO点 =====
interface IOPoint {
  tag: string                // 如 "LT-1001"
  description: string        // 如 "锅炉汽包液位"
  signalType: 'AI' | 'AO' | 'DI' | 'DO'
  unit: string               // 如 "mm" "%"
  range?: { low: number; high: number }
  pageId: string             // 所在页面
  boundPortId: string | null // 绑定到的功能块端口
}

// ===== 连线 =====
interface Connection {
  connectionId: string
  sourceBlockId: string
  sourcePortId: string
  targetBlockId: string
  targetPortId: string
}
```

## 功能一：页间跳转智能导航

### 功能描述

在组态画布中，用户经常需要追踪一个信号从哪里来、到哪里去，而信号源头或终点往往在另一张组态页面上。本功能提供两种导航方式：

1. **选中元素跳转**：用户选中一个功能块或IO点，系统分析其信号的上下游关系，列出关联的其他页面供一键跳转。
2. **搜索跳转**：用户在搜索框输入tag名或工艺关键词（如"汽包液位""给水泵"），系统模糊匹配并列出相关页面。

### 交互规范

#### 选中元素跳转

```
触发条件：用户单击选中一个功能块或IO点
展示位置：画布右侧弹出导航面板（侧边栏），不遮挡画布主体
面板内容：
  ┌─────────────────────────────────┐
  │ 📍 关联页面                    ✕ │
  │─────────────────────────────────│
  │ 🔴 直接关联（信号直连）          │
  │  ├ 主蒸汽温度控制 (P-001)       │
  │  │  └ 信号 TT-1001 → PID输入    │
  │  ├ 汽包液位保护 (P-005)         │
  │  │  └ 信号 LT-1001 来源定义     │
  │                                  │
  │ 🟡 同回路关联                    │
  │  ├ 给水阀控制 (P-013)           │
  │  │  └ 同属 给水系统              │
  │                                  │
  │ 🔵 相关页面                      │
  │  ├ 锅炉总貌 (P-000)             │
  │─────────────────────────────────│
  │ 点击页面名即可跳转               │
  └─────────────────────────────────┘

跳转行为：
  - 单击页面名 → 切换到目标页面，自动平移画布使关联元素居中
  - 目标页面上的关联元素高亮闪烁3秒（黄色边框脉冲动画）
  - 支持浏览器式前进/后退导航（维护跳转历史栈）
```

#### 搜索跳转

```
触发条件：用户点击顶部搜索框或按 Ctrl+K 快捷键
展示形式：居中弹出的搜索面板（类似VS Code的命令面板样式）
搜索逻辑：
  - 输入内容实时匹配（debounce 200ms）
  - 匹配范围：tag名、IO点描述、页面名称、功能块label
  - 结果按匹配度排序，显示匹配来源（如 "LT-1001 锅炉汽包液位 → 页面: 汽包液位保护"）
  - 回车或点击 → 跳转到目标页面并定位到匹配元素
```

### 核心实现逻辑

```
关联页面的查找算法：

输入：当前选中元素 selectedElement
输出：关联页面列表 RelatedPage[]

步骤：
1. 【直接关联】追踪选中元素所有端口的信号连接：
   - 遍历 selectedElement 的每个 port
   - 如果 port.connectedTo 指向的目标在其他 pageId 上 → 加入直接关联
   - 如果 selectedElement 是 IOPoint，查找 globalIOList 中同 tag 在其他页面的引用 → 加入直接关联
   
2. 【同回路关联】根据tag命名规则提取回路标识：
   - 提取 tag 前缀（如 "LT-1001" → 前缀 "1001" 或按项目规则提取区域号）
   - 在 pages 索引中查找 tags 数组包含相同前缀的页面 → 加入同回路关联
   - 排除当前页面和已在直接关联中的页面

3. 【相关页面】按工艺区域匹配：
   - 查找 area 字段相同的其他页面 → 加入相关页面
   - 排除已在上两类中的页面

4. 每类内部按关联信号数量降序排序
5. 总数上限 10 条
```

### 边界条件

- 当前页面无任何跨页信号时：导航面板显示"当前页面无跨页关联"，下方仍显示搜索框
- 跳转目标页面不存在（已被删除）：Toast提示"目标页面已删除"
- 跳转历史栈上限 50 条，超出后丢弃最早的记录
- 搜索输入为空时：显示最近访问的 5 个页面

### 验收标准

| 编号 | 测试项 | 操作步骤 | 预期结果 |
|------|--------|----------|----------|
| N-01 | 直连信号跳转 | 选中一个输入信号来自其他页面的功能块 | 导航面板"直接关联"分类下列出该信号所在页面 |
| N-02 | 跳转后定位 | 点击导航面板中的页面名 | 切换到目标页面，关联元素居中并高亮闪烁 |
| N-03 | 前进/后退 | 连续跳转3次后点击后退 | 依次返回上一页面，前进可恢复 |
| N-04 | 搜索tag名 | 搜索框输入 "LT-1001" | 结果列出包含该tag的所有页面 |
| N-05 | 搜索中文描述 | 搜索框输入 "汽包液位" | 结果列出描述匹配的IO点及所在页面 |
| N-06 | 模糊搜索 | 搜索框输入 "给水" | 匹配到 "给水泵联锁" "给水阀控制" 等页面 |
| N-07 | 空关联处理 | 选中一个无跨页信号的独立功能块 | 面板显示提示信息，搜索框仍可用 |

| N-10 | 搜索防抖 | 快速连续输入字符 | 不会每次输入都触发搜索，200ms防抖后触发一次 |

## 功能二：IO点连接自动推荐

### 功能描述

用户从功能块端口拖出连线时，系统根据端口的信号类型、工艺上下文和当前页面已有连接模式，推荐最可能的连接目标。推荐结果以浮动面板形式呈现在连线末端附近，同时在画布上高亮兼容的目标端口。

### 交互规范

```
触发条件：用户从一个端口开始拖线，拖出距离 > 20px 时触发推荐
展示时机：拖线开始后 300ms（防止误触）
展示位置：跟随鼠标指针，在指针右下方显示浮动推荐面板

推荐面板：
  ┌──────────────────────────────────────┐
  │ 🔍 过滤...                           │
  │──────────────────────────────────────│
  │ ⭐ LT-1001.PV  锅炉汽包液位   AI     │
  │    PSH-2003.OUT 高压报警开关   DI     │
  │    TT-1002.PV   过热蒸汽温度   AI     │
  │ ── 已连接 ──────────────────────      │
  │    FT-1005.PV   给水流量 (已占用) AI  │
  │──────────────────────────────────────│
  │ ↵ 选择  ESC 取消                      │
  └──────────────────────────────────────┘

画布上同步行为：
  - 当前页面上所有类型兼容且未连接的端口：绿色光晕高亮
  - 类型兼容但已连接的端口：灰色虚线高亮
  - 类型不兼容的端口：无变化（不干扰视觉）
  - 推荐面板中当前hover的条目对应的端口：放大高亮+连线预览虚线

连接完成：
  - 用户在面板中选择一个目标（点击或回车）→ 自动创建连线
  - 用户直接把线拖到某个端口上释放 → 直接连接（不依赖推荐面板）
  - 推荐面板在连线完成或按 ESC 后关闭

过滤框行为：
  - 输入文字后按 tag名 和 description 过滤推荐列表
  - 过滤不改变推荐排序，只隐藏不匹配的条目
  - 清空过滤框恢复完整列表
```

### 核心实现逻辑

```
推荐算法：

输入：sourcePort（拖线起始端口）
输出：推荐列表 Suggestion[]

步骤：
1. 【硬过滤】从当前页面和全局IO表中筛选候选目标：
   - sourcePort.direction === 'output' → 只匹配 direction === 'input' 的端口（反之亦然）
   - dataType 兼容检查：float ↔ float, bool ↔ bool（不允许 float → bool 直连）
   - signalType 兼容检查：
     * AI 端口 → 可连 AI/INTERNAL
     * AO 端口 → 可连 AO/INTERNAL  
     * DI 端口 → 可连 DI/INTERNAL
     * DO 端口 → 可连 DO/INTERNAL
     * INTERNAL → 可连任何同 dataType 的端口

2. 【评分排序】对通过硬过滤的候选端口评分：
   - 同功能块加分 +5（如PID的SP和PV通常需要连接）
   - 同页面加分 +3（当前页面内的端口优先）
   - tag前缀相同加分 +4（如 LT-1001 和 LIC-1001 属于同一回路）
   - 未连接加分 +2（已连接的降低优先级）
   - 常见连接模式加分 +3：
     * PID.OUT → AO（常见：PID输出连执行机构）
     * COMPARATOR.OUT → AND/OR.IN（常见：比较器输出连逻辑门）
     * AI → PID.PV（常见：测量值送PID）
     * DI → AND/OR.IN（常见：开关量送联锁逻辑）

3. 按分数降序排列，已连接的端口放到列表末尾并标注"已占用"
4. 返回 Top 10
```

### 边界条件

- 当前页面无兼容端口：面板显示"当前页面无可用目标"，提供"查看其他页面"链接
- 所有兼容端口都已连接：面板正常显示但全部标注"已占用"，用户仍可选择（覆盖原连接需二次确认）
- 拖线过程中页面缩放/平移：推荐面板跟随鼠标位置更新
- 同一端口被重复连接：弹出确认对话框"该端口已连接到 XXX，是否替换？"
- 拖线起点是未定义类型的端口（INTERNAL）：只按 dataType 过滤，不限制 signalType

### 验收标准

| 编号 | 测试项 | 操作步骤 | 预期结果 |
|------|--------|----------|----------|
| IO-01 | 推荐面板弹出 | 从PID.OUT端口拖出连线 | 300ms后出现推荐面板，列出AO类型的可用端口 |
| IO-02 | 类型过滤正确 | 从DI端口拖出连线 | 推荐列表中无AI/AO类型端口 |
| IO-03 | 未连接优先 | 页面上有已连接和未连接的兼容端口 | 未连接端口排在前面 |
| IO-04 | 画布高亮 | 拖线触发推荐 | 兼容端口绿色高亮，已连接端口灰色高亮 |
| IO-05 | 面板选择连接 | 在推荐面板中点击一个目标 | 自动创建连线，面板关闭 |
| IO-06 | 直接拖拽连接 | 忽略面板，直接把线拖到目标端口 | 正常连接，面板关闭 |
| IO-07 | 过滤功能 | 在面板过滤框输入tag片段 | 列表只显示匹配条目 |
| IO-08 | 已占用覆盖确认 | 选择一个已占用的端口 | 弹出确认对话框，确认后替换原连接 |
| IO-09 | ESC取消 | 拖线过程中按ESC | 取消拖线，面板关闭，画布高亮消失 |
| IO-10 | 无可用目标 | 在只有一个功能块的空页面从输出端口拖线 | 面板提示"当前页面无可用目标" |
| IO-11 | hover预览 | 在面板中hover某个推荐条目 | 画布上该端口放大高亮，显示虚线连线预览 |

## 功能三：组态逻辑错误检测

### 功能描述

用户完成组态编辑后，可触发逻辑检查，系统遍历当前页面（或选定范围）的功能块和连线，检测结构性错误、类型错误和工艺合理性问题。结果在底部面板分级展示，点击可定位到问题元素。

### 检测规则

```
===== ERROR级（红色，必须修复）=====

E1 悬空输入
   条件：功能块的 input 端口 connectedTo === null
   排除：端口有默认值配置的除外
   消息："{blockId} 的端口 {portName} 未连接输入信号"

E2 类型不匹配
   条件：连线两端的 dataType 不一致（如 float → bool）
   消息："{sourceTag}.{sourcePort} (float) 连接到 {targetTag}.{targetPort} (bool)，类型不兼容"

E3 循环依赖（无延时环路）
   条件：从某功能块出发沿连线可以回到自身，且路径上无 TIMER 类型功能块
   检测：对当前页面的连接关系做有向图环路检测（DFS）
   排除：路径上包含 TIMER/DELAY 类型功能块的环路（这是合法的反馈控制）
   消息："检测到无延时循环依赖：{blockId1} → {blockId2} → ... → {blockId1}"

E4 输出冲突
   条件：同一个 input 端口被两条以上连线连接
   消息："{targetBlockId}.{targetPort} 被多个信号源同时连接"

E5 信号方向错误
   条件：连线从 input 端口连到 input 端口，或从 output 连到 output
   消息："{portA} 和 {portB} 方向相同，无法建立有效连线"

===== WARNING级（黄色，建议修复）=====

W1 未使用输出
   条件：功能块的 output 端口 connectedTo === null 且不是页面的最终输出
   消息："{blockId} 的输出端口 {portName} 未被使用"

W2 量程未配置
   条件：AI/AO 类型的 IOPoint 的 range 字段为空
   消息："{tag} 未配置量程范围"

W3 联锁缺少旁路
   条件：连接到 DO 输出（执行机构）的信号路径中，无 bypass/旁路 类型功能块
   检测：从 DO 输出反向追踪信号路径，检查路径上有无 BYPASS/SWITCH/SELECTOR 类型块
   消息："{doTag} 的联锁逻辑链中未发现旁路功能块"

W4 开关量无防抖
   条件：DI 信号直接连接到 AND/OR 等逻辑门，中间无 TIMER/DEBOUNCE 功能块
   消息："{diTag} 直接连接逻辑运算，建议添加防抖延时"

W5 孤岛功能块
   条件：功能块所有端口的 connectedTo 均为 null（完全无连接）
   消息："{blockId} 与其他元素无任何连接"

===== INFO级（蓝色，优化建议）=====

I1 命名不规范
   条件：tag 不符合正则 /^[A-Z]{2,4}-\d{3,5}$/ （可配置）
   消息："{tag} 命名不符合项目规范"

I2 冗余连线
   条件：两个端口之间存在多条连线
   消息："{portA} 到 {portB} 之间存在重复连线"
```

### 交互规范

```
触发方式：
  1. 工具栏 "检查" 按钮（图标：盾牌 + 对勾）
  2. 快捷键 Ctrl+Shift+C
  3. 保存时自动后台检查（结果缓存，不重复弹出）

检查范围：
  - 默认检查当前页面
  - 可选"检查全部页面"（耗时较长，显示进度条）

结果面板（画布底部，可拖拽调整高度）：
  ┌──────────────────────────────────────────────────────────┐
  │ 检查结果    ❌ 3 Errors   ⚠️ 5 Warnings   ℹ️ 2 Infos     │
  │──────────────────────────────────────────────────────────│
  │ ❌ E1  FB-0042 的端口 IN2 未连接输入信号        [定位] [忽略] │
  │ ❌ E3  检测到循环依赖: FB-012→FB-015→FB-012    [定位] [忽略] │
  │ ❌ E4  FB-0099.IN1 被多个信号源同时连接         [定位] [忽略] │
  │ ⚠️ W1  FB-0042 的输出端口 OUT2 未被使用         [定位] [忽略] │
  │ ⚠️ W2  LT-1001 未配置量程范围                   [定位] [忽略] │
  │ ...                                                       │
  │──────────────────────────────────────────────────────────│
  │ 上次检查: 14:32:05  检查耗时: 0.3s  已忽略: 1              │
  └──────────────────────────────────────────────────────────┘

[定位] 按钮行为：
  - 点击后画布自动平移+缩放，使问题元素居中
  - 问题元素红色虚线边框闪烁3秒
  - 如果涉及连线问题（如E2类型不匹配），高亮整条连线为红色
  - 如果涉及环路（E3），高亮环路上所有功能块和连线

[忽略] 按钮行为：
  - 该条结果标记为灰色删除线
  - 下次检查不再报告相同位置的相同问题
  - 忽略列表可在设置中管理（查看/恢复）

面板行为：
  - 可按级别过滤（只看Error / 只看Warning）
  - 可按功能块ID排序
  - 面板可最小化为底部状态栏（只显示计数）
  - 双击结果行 = 点击 [定位]
```

### 核心实现逻辑

```
检查流程（伪代码）：

function checkPage(page: ConfigPage): CheckResult[] {
  const results: CheckResult[] = []
  const graph = buildDirectedGraph(page.connections)
  
  // E1: 悬空输入
  for (const block of page.blocks) {
    for (const port of block.ports) {
      if (port.direction === 'input' && !port.connectedTo && !port.hasDefault) {
        results.push({ code: 'E1', severity: 'error', elements: [block.blockId], ... })
      }
    }
  }
  
  // E2: 类型不匹配
  for (const conn of page.connections) {
    const sourcePort = findPort(conn.sourcePortId)
    const targetPort = findPort(conn.targetPortId)
    if (sourcePort.dataType !== targetPort.dataType) {
      results.push({ code: 'E2', severity: 'error', elements: [conn.connectionId], ... })
    }
  }
  
  // E3: 循环依赖
  const cycles = detectCycles(graph) // 标准DFS环路检测
  for (const cycle of cycles) {
    if (!cycle.some(blockId => getBlockType(blockId) === 'TIMER')) {
      results.push({ code: 'E3', severity: 'error', elements: cycle, ... })
    }
  }
  
  // E4: 输出冲突
  const inputConnectionCount = countConnectionsPerInputPort(page.connections)
  for (const [portId, count] of inputConnectionCount) {
    if (count > 1) {
      results.push({ code: 'E4', severity: 'error', elements: [portId], ... })
    }
  }
  
  // E5, W1-W5, I1-I2 类似逻辑...
  
  return results.sort(bySeverityThenCode)
}
```

### 边界条件

- 空页面（无功能块）：显示"当前页面为空，无需检查"
- 大页面（>200个功能块）：显示进度条，分批检查，渐进渲染结果
- 全部检查通过（0 errors, 0 warnings, 0 infos）：面板显示绿色"✅ 检查通过，未发现问题"
- 用户在检查过程中编辑了页面：检查结果标记为"已过期（页面已修改）"，提示重新检查
- 忽略列表持久化：存储在项目配置中，换电脑后仍然生效
