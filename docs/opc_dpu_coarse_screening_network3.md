# LH3 OPC DPU 粗筛清单(network3)

> 生成时间:2026-06-14  
> 项目名称:LH3  
> 来源:当前 NT6000 工程目录 `demo/network3` 的 DPU 点表。  
> 用途:新工程开始阶段,先按 DPU 所属总体系统粗筛哪些 DPU 需要纳入 OPC 通讯;后续再按点级规则细筛。

## 方法说明

- 以 `%NT6000_HOME%\project\demo\network\network3\DPU\DPUxxxx\tag.csv` 为主,并已同步处理 `temp` / `target` 两份点表。注意:这里的 `demo` 是 NT6000 当前磁盘工程目录名,业务项目名按 `LH3` 记录。
- 当前点表已做过基础清理:只保留 `测点名称` 以 `HW` 开头的硬件 IO 点,删除描述为空和描述含“备用”的点,`OPC输出` 统一置为 `Y`。
- “硬件 IO 点数”是当前清理后保留的 `HW` 点数。
- 系统归属依据为点表 `描述`、`设计编号(KKS)` 前缀和典型测点名称;这是粗筛判断,不是最终系统边界。
- `硬件 IO 点数=0` 的 DPU 当前没有可直接纳入 OPC 的硬件 IO 点;如确需纳入软点/组态点,需要单独讨论。

## 项目内点表版本

LH3 在仓库内统一使用三层点表目录:

```text
projects/lh3/io/raw/       # 原始版本,只作备份/追溯
projects/lh3/io/filtered/  # 筛选版本,对应本文件记录的粗筛流程
projects/lh3/io/simple/    # 精简版本,后续细筛后的最终点表,/script 读取这里
```

`projects/lh3/project.yaml` 中 `io_dir` 指向 `projects/lh3/io/simple`,`io_full_dir` 指向 `projects/lh3/io/filtered`。不再使用 `simple/简化/`。

## 粗筛提示

- 需要做 CCS/锅炉主控闭环时,通常优先关注 `DPU3012`、`DPU3013`、`DPU3010`、`DPU3002`~`DPU3009`。
- 需要做燃烧/FSSS/制粉相关联锁时,通常优先关注 `DPU3001`~`DPU3009`。
- 需要做给水/汽泵/凝结水相关时,通常关注 `DPU3025`~`DPU3033`。
- 需要做 MEH/DEH/ETS 或汽机接口时,通常关注 `DPU3037`、`DPU3041`~`DPU3044`。
- 需要做环保/脱硝/脱硫/除尘时,通常关注 `DPU3018`、`DPU3019`、`DPU3048`~`DPU3055`。
- `DPU3020`、`DPU3022`、`DPU3023`、`DPU3024`、`DPU3039`、`DPU3040` 当前没有保留下来的硬件 IO 点,粗筛时可先排除;若后续要纳入 SH/软点,需要另开规则。
